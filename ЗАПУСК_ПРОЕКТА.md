## 6. Push Notification Service (MPNS)

### Требует Docker
```bash
# Убедись, что Docker Desktop запущен

cd /Users/devbroseph/Mattermost/mpns

# Запуск MPNS
docker compose up -d

# Настройка Mattermost на работу с MPNS
bash setup_mpns.sh ВАШ_ADMIN_EMAIL ВАШ_ADMIN_ПАРОЛЬ
```

### Перед запуском — добавь ключи в `mpns/config.json`:

**iOS (APNs):**
1. [developer.apple.com](https://developer.apple.com) → Certificates → Push Notifications
2. Bundle ID: `ru.obrazcova.academy`
3. Скачай `.pem` файл, укажи путь в `ApplePushCertPrivate`

**Android (FCM):**
1. [console.firebase.google.com](https://console.firebase.google.com) → Настройки → Cloud Messaging
2. Скопируй Server Key в `AndroidApiKey`

### Перезапуск после изменения конфига
```bash
cd /Users/devbroseph/Mattermost/mpns
docker compose restart
```

### Просмотр логов MPNS
```bash
cd /Users/devbroseph/Mattermost/mpns
docker compose logs -f
```

## 8. Деплой на сервер (продакшн)

### Требования к серверу
- Ubuntu 22.04 LTS / CentOS 8+
- 4 GB RAM minimum
- PostgreSQL 15
- Nginx (reverse proxy)
- SSL-сертификат (Let's Encrypt)

### Базовый деплой
```bash
# На сервере:
# 1. Скачать бинарник Mattermost
wget https://releases.mattermost.com/10.5.1/mattermost-10.5.1-linux-amd64.tar.gz
tar -xzf mattermost-10.5.1-linux-amd64.tar.gz

# 2. Настроить config/config.json (DSN базы данных, SiteURL)

# 3. Запустить как systemd-сервис
sudo cp /opt/mattermost/bin/mattermost /etc/systemd/system/
sudo systemctl enable mattermost
sudo systemctl start mattermost

# 4. Nginx конфиг (proxy_pass http://localhost:8065)
# 5. Certbot для SSL
sudo certbot --nginx -d your-domain.ru
```

## SMS Auth Service (sms.ru) — один раз

```bash
# 1. Установить зависимости
cd /Users/devbroseph/Mattermost/sms_auth
npm install

# 2. Настроить сервис (создаёт .env с токеном Mattermost)
bash /Users/devbroseph/Mattermost/sms_auth/setup.sh admin@academy.ru AdminPass123!

# 3. Вставить реальный API-ключ sms.ru в .env:
#    Откройте: /Users/devbroseph/Mattermost/sms_auth/.env
#    Замените: SMSRU_API_ID=ВСТАВЬТЕ_ВАШЕ_API_ID
#    API-ключ: https://sms.ru → Настройки → API

# 4. Переключить в боевой режим:
#    Измените: SMSRU_TEST=0  (или оставьте 1 для разработки)

# 5. Запустить:
npm start
# → Доступен на http://localhost:3002
```

### Привязка телефонов пользователей

```bash
# После запуска сервиса — привяжите телефоны к аккаунтам:
# (используйте токен из .env → MM_ADMIN_TOKEN)

# Студент:
curl -X POST http://localhost:3002/api/admin/phone-users \
  -H "Authorization: Bearer <MM_ADMIN_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"phone": "+79161234501", "email": "student@academy.ru"}'

# Педагог:
curl -X POST http://localhost:3002/api/admin/phone-users \
  -H "Authorization: Bearer <MM_ADMIN_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"phone": "+79161234502", "email": "teacher@academy.ru"}'

# Или через Admin-панель в приложении:
# Профиль → Управление пользователями → тап на пользователя → «Привязать телефон»
```

### Баланс sms.ru

```bash
curl http://localhost:3002/api/admin/balance \
  -H "Authorization: Bearer <MM_ADMIN_TOKEN>"
```
