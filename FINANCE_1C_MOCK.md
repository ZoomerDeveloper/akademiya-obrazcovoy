# Финансы и 1С — контракт для будущей интеграции (черновик)

Цель ТЗ: раздел «Мои оплаты», напоминания об оплате, при необходимости — выгрузка из 1С.

## Рекомендуемый минимальный API (мок или бэкенд Академии)

`GET /api/finance/me?user_id=...` или по Mattermost user id + токен сессии.

Ответ (пример):

```json
{
  "user_id": "mattermost-user-id",
  "items": [
    {
      "id": "inv-001",
      "title": "Обучение, февраль 2026",
      "amount": 25000,
      "currency": "RUB",
      "due_date": "2026-02-25",
      "status": "pending|paid|overdue",
      "pay_url": "https://..."
    }
  ]
}
```

## Интеграция с 1С
- Периодическая выгрузка (файл / HTTP-сервис 1С) → промежуточная БД или сервис → отдача в веб/мобилу.
- Идентификация пользователя: email или внешний `mm_user_id`, заведённый в 1С.

## Текущее состояние проекта
- В вебе блок «Мои оплаты» опирается на `payment_link` и статусы из **booking_service** (не 1С).
- Для полноценного ТЗ нужен отдельный микросервис или расширение API с авторизацией по Mattermost.
